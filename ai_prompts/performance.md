Проведи ревью и оптимизацию проекта с фокусом на performance, best practices, сложность алгоритмов, читаемость и порядок кода.

Цель:

- улучшить производительность;
- убрать лишние ререндеры;
- привести код к чистому, читаемому и поддерживаемому виду;
- проверить алгоритмическую сложность ключевых операций;
- не сломать бизнес-логику и архитектуру.

Проверь:

1. Performance:

- нет ли лишних ререндеров React-компонентов;
- используются ли `memo`, `useMemo`, `useCallback` только там, где это оправдано;
- оптимизирован ли `FlatList`;
- есть ли `keyExtractor`;
- не создаются ли тяжёлые inline-объекты и inline-функции без необходимости;
- не выполняется ли сортировка списка прямо в render;
- не генерируются ли FakerJS-данные повторно;
- не выполняются ли повторные сетевые запросы;
- не блокируется ли UI тяжёлыми вычислениями.

2. Алгоритмическая сложность:

- оцени сложность основных операций через Big O:
  - загрузка списка;
  - сортировка избранных постов;
  - поиск поста по id;
  - toggle favorite;
  - получение details;
- если где-то используется неэффективный `O(n)` поиск там, где лучше `Map`, `Set` или normalized state, предложи улучшение;
- избранные id желательно хранить как `Set` или объект-словарь для `O(1)` проверки;
- details желательно хранить по id для `O(1)` доступа;
- сортировку списка вынести в selector и мемоизировать.

3. Best practices:

- соблюдай TypeScript strict-подход;
- избегай `any`;
- используй явные типы для API, store, navigation params и repositories;
- не смешивай UI, business logic, data layer и persistence;
- соблюдай SOLID, DRY, KISS, YAGNI, SoC;
- не добавляй новые зависимости без необходимости;
- не меняй публичное поведение приложения.

4. Читаемость:

- улучши имена переменных, функций и типов;
- разнеси слишком большие функции;
- убери дублирование;
- удали мёртвый код;
- убери неиспользуемые импорты;
- сделай код линейным и понятным;
- сложную логику вынеси в отдельные функции/selectors/helpers.

5. Порядок в проекте:

- проверь структуру папок;
- проверь порядок импортов;
- сгруппируй импорты:
  1. React/React Native;
  2. внешние библиотеки;
  3. navigation/store/services/repositories;
  4. shared/entities/types;
  5. локальные компоненты/styles;
- проверь единый стиль именования файлов;
- проверь, что компоненты, hooks, services, repositories и types лежат в правильных местах;
- не перемещай файлы без необходимости, если это ухудшит читаемость diff.

6. Документация:

- создай или обнови файл:
  `ai_documentation/12_performance_and_code_quality_review.md`

В документе укажи:

- найденные performance-проблемы;
- найденные проблемы читаемости;
- Big O для ключевых операций;
- что было улучшено;
- какие решения оставлены без изменений и почему;
- какие проверки нужно выполнить вручную.

Правила:

- Сначала проведи аудит.
- Затем внеси только безопасные улучшения.
- Не переписывай приложение полностью.
- Не меняй бизнес-требования.
- Не ломай persistence.
- Не ломай one-time fetch.
- Не ломай избранное.
- Не ломай FakerJS-кэширование.
- После изменений запусти TypeScript/линтер, если команды есть в `package.json`.

После завершения выведи краткий отчёт:

1. Что было оптимизировано.
2. Какие файлы изменены.
3. Какая сложность ключевых операций до/после.
4. Какие best practices применены.
5. Что нужно проверить вручную.
