# 07. Стратегия тестирования

## Ручное тестирование
Главная стратегия для тестового задания — ручная проверка ключевого пользовательского flow на iOS/Android.

### Сценарий: первый запуск
1. Очистить данные приложения.
2. Запустить приложение.
3. Убедиться, что PostsScreen показывает loading.
4. Дождаться списка постов.
5. Проверить наличие title/body и 32x32 image.
6. Убедиться, что данные сохранены в storage.

### Сценарий: повторный запуск
1. Перезапустить приложение.
2. Убедиться, что список появляется из cache.
3. Убедиться, что изображения не изменились.
4. Проверить отсутствие повторного network-запроса при наличии cache.

### Сценарий: добавление в избранное
1. Открыть details поста.
2. Нажать кнопку добавления в избранное.
3. Вернуться на список.
4. Проверить визуальное выделение и поднятие поста вверх.
5. Перезапустить приложение и проверить сохранение favorite state.

### Сценарий: удаление из избранного
1. Открыть details избранного поста.
2. Нажать кнопку удаления.
3. Вернуться на список.
4. Проверить, что visual state снят и сортировка обновилась.

### Сценарий: DetailsScreen
1. Нажать на пост.
2. Проверить загрузку details из `/posts/{id}`.
3. Проверить 300x300 image.
4. Повторно открыть тот же пост и убедиться, что данные берутся из cache.

## Unit-тесты, которые можно добавить
- Mapper/factory API post -> enriched post.
- Selector сортировки избранных.
- Toggle favorite reducer/action logic.
- Storage adapter serialization/deserialization.

## Integration-тесты, которые можно добавить
- Repository: cache miss -> API -> enrich -> save -> return.
- Repository: cache hit -> no API call.
- Store: loadPosts восстанавливает данные.
- Store: toggleFavorite сохраняет `favoriteIds`.

## Проверка отсутствия повторных запросов
- Замокать API client в тестах repository.
- Сначала вернуть cache и проверить, что API client не вызван.
- Для ручной проверки можно временно логировать network-вызовы или использовать debugger/network inspector.

## Проверка стабильности FakerJS
- Сохранить URL изображения после первого запуска.
- Перезапустить приложение.
- Сравнить URL.
- Повторно открыть details и проверить, что 300x300 URL не изменился.
