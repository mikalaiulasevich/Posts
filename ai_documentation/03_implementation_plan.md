# 03. План реализации

## Фаза 0. Анализ требований и подготовка документации

Цель: понять задачу, выявить риски, выбрать архитектуру и подготовить план.

Итерации:

- 0.1 Анализ требований.
- 0.2 Проектирование архитектуры.
- 0.3 Проектирование модели данных.
- 0.4 Проектирование persistence-стратегии.
- 0.5 Подготовка AI-документации.

Ожидаемый результат: создана папка `ai_documentation`, подготовлен `AGENTS.MD`, код приложения не изменён.

## Фаза 1. Инициализация проекта

Цель: подготовить основу приложения.

Итерации:

- 1.1 Проверить зависимости в `package.json`.
- 1.2 Добавить `@faker-js/faker`, если он ещё отсутствует.

Критерий завершения: проект компилируется после добавления базовых файлов.

## Фаза 2. Data layer

Цель: реализовать API, repositories, storage adapter и one-time fetch.

Итерации:

- 2.1 Типизация API-ответов.
- 2.2 API-клиент для списка постов.
- 2.3 API-клиент для details.
- 2.4 MMKV storage adapter.
- 2.5 `PostsRepository`.
- 2.6 `DetailsRepository`.
- 2.7 Генерация и сохранение FakerJS-изображений.

Критерий завершения: repositories возвращают enriched данные и повторно используют cache.

## Фаза 3. State layer

Цель: реализовать глобальное состояние и избранное.

Итерации:

- 3.1 Store shape.
- 3.2 Actions для загрузки списка.
- 3.3 Actions для загрузки details.
- 3.4 Action `toggleFavorite`.
- 3.5 Selectors для сортировки избранных.

Критерий завершения: состояние восстанавливается после перезапуска, favorite меняет порядок списка.

## Фаза 4. Navigation
