# 00. Анализ задачи

## Краткое описание
Нужно разработать небольшое мобильное приложение на React Native CLI для iOS и Android: список постов, экран деталей и избранное. На текущем шаге выполняется только анализ требований и подготовка AI-документации; код приложения не реализуется.

## Источники требований
- `task.md` — исходное тестовое задание.
- `prompt.md` — правила AI-only процесса и набор документов для подготовки.
- `package.json` — текущее состояние проекта и уже добавленные зависимости.

## Функциональные требования
1. `PostsScreen`:
   - загрузить список из `https://jsonplaceholder.typicode.com/posts`;
   - обогатить каждый пост изображением 32x32 через FakerJS;
   - загрузить список только один раз;
   - сохранить список и сгенерированные изображения локально;
   - при повторном запуске не делать повторный запрос, если данные уже сохранены;
   - визуально выделять избранные посты;
   - поднимать избранные посты вверх списка.
2. `DetailsScreen`:
   - открывать пост по нажатию из списка;
   - загрузить данные из `https://jsonplaceholder.typicode.com/posts/{id}`;
   - обогатить детальный пост изображением 300x300 через FakerJS;
   - загружать детали каждого поста только один раз;
   - сохранить детальные данные и изображение локально;
   - добавить динамическую кнопку toggle favorite.
3. Избранное:
   - состояние избранного глобальное;
   - изменение на `DetailsScreen` сразу влияет на порядок и визуальное состояние на `PostsScreen`;
   - состояние переживает перезапуск приложения.

## Технические требования
- React Native `>=0.77`, без Expo.
- TypeScript.
- React Navigation `>=7`.
- FakerJS.
- Обязательный state-manager.
- Чистая архитектура: разделение UI, состояния, бизнес-логики и data layer.
- Repository Pattern для доступа к данным.
- Локальное persistent storage.
- GitHub-репозиторий с README и инструкциями запуска.
- AI-only доказательства: промпты, правила, чат/выгрузки или скриншоты.

## Текущее состояние репозитория
- React Native: `0.86.0` — требование `>=0.77` выполнено.
- React Navigation: установлены `@react-navigation/native` и `@react-navigation/native-stack` версии 7.x.
- State-manager: установлен `zustand`.
- Storage: установлен `react-native-mmkv` и `react-native-nitro-modules`.
- FakerJS: `@faker-js/faker` пока отсутствует в `package.json`; это риск для фазы реализации.
- `src/` пустая, приложение ещё не реализовано.
- README пока стандартный от React Native CLI.

## Скрытые требования и риски
- One-time fetch должен быть проверяемым: нужно явно разделить сетевой источник и локальный cache source.
- FakerJS-картинки нельзя генерировать в рендере, иначе они будут меняться.
- Details-картинка 300x300 и list-картинка 32x32 должны храниться отдельно.
- Если список уже сохранён, приложение не должно повторно обращаться к API даже после перезапуска.
- Нужно корректно обработать loading/error/empty состояния.
- MMKV — native dependency; перед iOS запуском может понадобиться `bundle exec pod install`.
- FakerJS отсутствует в зависимостях: на фазе реализации нужно добавить `@faker-js/faker`.
- Проверяющий ожидает простоту запуска: README должен быть практичным, не академическим.

## Критерии готовности приложения
- Приложение запускается на iOS и Android.
- Первый запуск загружает и сохраняет список.
- Повторный запуск использует локальные данные.
- Детали каждого поста загружаются один раз и затем берутся из storage.
- Избранное переключается, сохраняется и отражается в списке.
- Избранные посты визуально выделены и расположены сверху.
- FakerJS-изображения стабильны после генерации.
- README содержит команды установки и запуска.
- AI-документация приложена к репозиторию.
