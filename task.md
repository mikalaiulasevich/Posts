Тестовое задание: Разработка мобильного приложения на React Native

Цель:
Разработать небольшое мобильное приложение для iOS и Android с использованием React Native(>=0.77), TypeScript, React Navigation(>=7), FakerJS.

Описание задачи:
Создать приложение для отображения списка постов. Каждый пост можно открыть в отдельном экране с возможностью их сохранения в избранное. Каждый избранный пост “поднимается” вверх списка.

Функционал:
Необходимо реализовать экран PostsScreen (рис.1). Список постов брать из ресурса https://jsonplaceholder.typicode.com/posts Также необходимо “обогатить” ответ изображением 32х32 из библиотеки FakerJS
 Необходимо реализовать экран DetailsScreen (рис.2). Пост брать из ресурса https://jsonplaceholder.typicode.com/posts/{id} Также необходимо “обогатить” ответ изображением 300х300 из библиотеки FakerJS
На экране DetailsScreen добавить динамическую кнопку-переключатель. При нажатии - добавляется в избранное, при повторном - удаляется из избранного
На экране PostsScreen необходимо отображать список постов с учетом их статуса избранного: если пост в избранном - в общем списке он визуально выделяется и оказывается вверху списка
Сгенерированный список запрашивается только 1 раз и никогда не обновляется. Добавленные картинки из FakerJS также добавляются только 1 раз - и для PostsScreen, и для DetailsScreen
При повторном открытии приложения данные сохраняются и повторно не запрашиваются. Все посты, добавленные в избранное - отмечены сразу

Требования к выполнению: 
Задание должно быть выполнено с использованием AI-агентов (AI-only подход). Вместе с реализованным заданием необходимо предоставить:
Скриншоты или выгрузку чатов из AI-агента/AI-чата
Промпты и правила, которые использовались для генерации решения
Технические требования:
Использование React Native (не Expo)
Использование TypeScript
Управление состоянием: используйте любой state-manager. Использование обязательно
Дизайн произвольный
Чистая архитектура (разделение UI, логики, состояния)
Код должен быть выложен в GitHub (репозиторий с README и инструкцией по запуску). Проверяющий должен установить проект одной командой, второй - запустить
