   - Список запрашивается только один раз.
   - После первого успешного запроса список должен сохраняться локально.

     - он визуально выделяется;
     - он поднимается вверх списка.
   - Все избранные посты должны быть отмечены сразу после повторного открытия приложения.

2. Экран DetailsScreen:
   - Открывается по нажатию на пост.
   - Пост нужно получить из:
     https://jsonplaceholder.typicode.com/posts/{id}
   - Пост нужно “обогатить” изображением 300x300 из FakerJS.
   - Детальная информация по каждому посту должна запрашиваться только один раз.
   - После первого успешного запроса детальные данные должны сохраняться локально.
   - FakerJS-картинка для DetailsScreen также должна генерироваться только один раз.
   - На экране должна быть динамическая кнопка-переключатель:
     - если пост не в избранном — кнопка добавляет его в избранное;
     - если пост уже в избранном — кнопка удаляет его из избранного.

Предпочтительные технические решения, которые нужно рассмотреть:

State-manager:
- Zustand.
Причина:
- простой;
- хорошо подходит для небольшого приложения;
- не требует большого boilerplate;
- легко подключить persistence;
- подходит для глобального состояния избранного и кэшированных постов.

Storage:
- AsyncStorage или MMKV.
Сравни их.
Для тестового задания предпочтительно выбрать AsyncStorage, если важна простота установки и проверки.
MMKV можно выбрать, если хочешь подчеркнуть производительность, но обязательно учти native setup.

API:
- fetch или axios.
Для тестового задания предпочтительно использовать fetch, чтобы уменьшить количество зависимостей.
Если выбираешь axios — обоснуй.

Architecture:
- Clean Architecture в упрощённом виде:
  - `src/shared`
  - `src/entities`
  - `src/features`
  - `src/screens`
  - `src/navigation`
  - `src/store`
  - `src/services`
  - `src/repositories`

Паттерны:
- Repository Pattern для доступа к данным.
- Adapter Pattern для storage.
- Selector pattern для derived state.
- Factory/helper для создания enriched posts.
- Single Responsibility Principle для разделения UI, store, repositories и services.
